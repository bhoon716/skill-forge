# my-new-skill

[English](./README.md)

이 스킬이 무엇을 하는지 짧게 설명합니다.

한두 문장으로 작성하세요. 단순히 주제를 설명하기보다, 어떤 워크플로를 수행하는지 설명하는 것이 좋습니다.

## 무엇을 하나요?

`my-new-skill`은 특정 작업을 반복 가능한 절차로 수행하도록 에이전트를 돕습니다.

이 섹션을 실제 스킬에 맞게 교체하세요.

좋은 설명:

> 이 스킬은 제품 아이디어를 검토하면서 대상 사용자, pain point, 성공 기준, 리스크, 가장 작은 다음 행동을 점검합니다.

약한 설명:

> 이 스킬은 제품 아이디어를 도와줍니다.

## 언제 사용하나요?

사용자가 다음을 요청할 때 이 스킬을 사용합니다.

- 특정 워크플로 적용
- 특정 종류의 입력 검토 또는 변환
- 구조화된 결과물 생성
- 반복 가능한 의사결정 절차 수행

## 언제 사용하지 않나요?

다음 경우에는 이 스킬을 사용하지 않습니다.

- 단순 사실 질문
- 일반적인 브레인스토밍
- 관련 없는 글쓰기 또는 코딩 작업
- 더 구체적인 다른 스킬이 적합한 요청

## 예시 프롬프트

켜져야 하는 경우:

```txt
이 스킬이 켜져야 하는 현실적인 사용자 요청을 작성하세요.
```

```txt
또 다른 trigger 예시를 작성하세요.
```

켜지면 안 되는 경우:

```txt
비슷해 보이지만 이 스킬이 켜지면 안 되는 요청을 작성하세요.
```

```txt
또 다른 non-trigger 예시를 작성하세요.
```

## 기대 동작

이 스킬이 사용되면 에이전트는 다음을 수행해야 합니다.

1. 작업 대상을 파악합니다.
2. 워크플로를 적용합니다.
3. 결과물이 품질 기준을 충족하는지 확인합니다.
4. 명확한 최종 결과를 반환합니다.

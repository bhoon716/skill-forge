# Naming Guide

Skill 이름은 다음 원칙을 따른다.

- 소문자, 숫자, 하이픈만 사용한다
- 짧고 기억하기 쉬워야 한다
- 동작을 대략 짐작할 수 있어야 한다
- 디렉터리 이름과 `SKILL.md`의 `name`은 같게 둔다

## Good examples

- `ultra-grill-me`
- `prd-shredder`
- `issue-splitter`
- `launch-checklist`
- `architecture-cross-examiner`

## Bad examples

- `UltraGrillMe`
- `my skill`
- `skill_v2_final`
- `The Ultimate Planning Assistant`

## Frontmatter

```md
---
name: ultra-grill-me
description: ...
---
```
